# Stage 4 — Metrics Adjustment Approval

## What you're approving
The decisions applied to every metric from Stage 3 so that the list we take into data collection is both **representative** of the expected value and **currently measurable**. For each metric, one of these decisions has been applied:

- **Keep as-is** — Feasibility 1 metric, no changes needed.
- **Activate** — Feasibility 2 metric, we'll set up extraction (preferably automated).
- **Replace with proxy** — Feasibility 3 metric, we've proposed a proxy that captures the same intent using available data.
- **Defer** — Metric only becomes measurable post-deployment; it's flagged for later implementation.
- **Eliminate** — Metric is redundant, unrepresentative, or has no viable proxy.

No STOP condition was triggered — all metrics are measurable.

Approving locks in the final list of metrics we'll use for the rest of the workflow — Stage 5 collects data for these specific metrics, Stage 6 calculates their business value, and Stage 7 reports on them.
